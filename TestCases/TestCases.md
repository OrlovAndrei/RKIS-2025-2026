# Тест-кейсы для TodoList Application

## Введение
Данный документ содержит набор тест-кейсов для проверки функциональности консольного приложения TodoList.  
Все скриншоты выполнения тестов находятся в папке `Screenshots`.

---

## 1. Пользователи и авторизация

### TC-01 Создание нового пользователя
**Описание:** Проверка успешного создания нового профиля.  
**Предусловия:** Приложение запущено, нет активного пользователя.  
**Последовательность действий:**
1. При запросе "Войти в существующий профиль? [y/n]" ввести `n`.
2. Ввести логин: `clashkatop`.
3. Ввести пароль: `sbzq`.
4. Ввести имя: `Gleb`.
5. Ввести фамилию: `Agulov`.
6. Ввести год рождения: `2007`.
**Ожидаемый результат:**  
- Сообщение "Профиль создан! Добро пожаловать, !"  
- Профиль сохраняется в файле `data/profile.csv`.  
**Скриншоты:**  
- `Screenshots/TC-01-step1.png`  

### TC-02 Авторизация существующего пользователя
**Описание:** Проверка входа в созданный ранее профиль.  
**Предусловия:** Существует пользователь с логином `clashkatop` и паролем `sbzq`.  
**Последовательность действий:**
1. При запросе выбрать `y`.
2. Ввести логин: `clashkatop`.
3. Ввести пароль: `sbzq`.
**Ожидаемый результат:**  
- Сообщение "вход выполнен!"  
- `CurrentProfileId` устанавливается для этого пользователя.  
**Скриншоты:** `Screenshots/TC-02-login.png`

### TC-03 Ошибка авторизации (неверный пароль)
**Описание:** Попытка входа с неверным паролем.  
**Предусловия:** Пользователь `clashkatop` существует.  
**Последовательность действий:**

2. Ввести логин: `clashkatop`.

### TC-04 Выход из профиля

## 2. Добавление задач (add)

### TC-07 Добавление простой задачи
**Описание:** Добавление задачи с обычным текстом.  
**Предусловия:** Пользователь авторизован.  
**Последовательность действий:**
1. Ввести команду `add поиграть в компьютер`.
**Ожидаемый результат:**  
- Сообщение "Задача добавлена: поиграть в компьютер".  
- Задача появляется в списке при `view`.  
**Скриншоты:** `Screenshots/TC-07-add-simple.png`

### TC-08 Добавление задачи с пустым названием
**Описание:** Попытка добавить задачу с пустым текстом.  
**Предусловия:** Авторизован.  
**Последовательность действий:**
1. Ввести команду `add`.
**Ожидаемый результат:**  
- Сообщение "недостаточно параметров. Использование: add <текст>" (исключение).  
**Скриншоты:** `Screenshots/TC-08-add-empty.png`

### TC-09 Добавление задачи с длинным текстом (>30 символов)
**Описание:** Добавление задачи, текст которой длиннее 30 символов.  
**Последовательность действий:**
1. `add Это очень длинный текст задачи, который должен быть обрезан в списке`
**Ожидаемый результат:**  
- В команде `view` текст обрезается до 24 символов + "...".  
- В `read` виден полный текст.  
**Скриншоты:** `Screenshots/TC-09-long-text.png`

### TC-10 Многострочная задача
**Описание:** Добавление задачи с флагом `--multiline` (`-m`).  
**Последовательность действий:**
1. `add -m`
2. Ввести первую строку.
3. Ввести вторую строку.
4. Ввести `!end`.
**Ожидаемый результат:**  
- Задача добавлена с текстом, содержащим переводы строк.  
- В `view` отображается первая строка (обрезанная).  
- В `read` видны все строки.  
**Скриншоты:** `Screenshots/TC-10-multiline.png`

### TC-11 Добавление задачи со служебными символами
**Описание:** Проверка корректного сохранения символов типа `;`, `"`, `\`, `/`.  
**Последовательность действий:**
1. `add Текст с ; и "кавычками" и \ слэшем /`
**Ожидаемый результат:**  
- Текст сохраняется и отображается корректно (экранирование при сохранении в CSV не ломает данные).  
**Скриншоты:** `Screenshots/TC-11-special-chars.png`

## 3. Просмотр задач (view)

### TC-12 Просмотр всех задач (без флагов)
**Описание:** Команда `view` без флагов.  
**Предусловия:** Есть несколько задач.  
**Последовательность действий:**
1. `view`
**Ожидаемый результат:**  
- Таблица с колонкой "Задача" (текст обрезан).  
**Скриншоты:** `Screenshots/TC-12-view-default.png`

### TC-13 Просмотр с флагом --index
**Описание:** `view -i` или `view --index`.  
**Ожидаемый результат:** Добавляется колонка с номером задачи.  
**Скриншоты:** `Screenshots/TC-13-view-index.png`

### TC-14 Просмотр с флагом --status
**Описание:** `view -s`.  
**Ожидаемый результат:** Добавляется колонка со статусом.  
**Скриншоты:** `Screenshots/TC-14-view-status.png`

### TC-15 Просмотр с флагом --date
**Описание:** `view -d`.  
**Ожидаемый результат:** Добавляется колонка с датой последнего изменения.  
**Скриншоты:** `Screenshots/TC-15-view-date.png`

### TC-16 Просмотр со всеми флагами
**Описание:** `view -a` или комбинация `-isd`.  
**Ожидаемый результат:** Все три колонки и задача.  
**Скриншоты:** `Screenshots/TC-16-view-all.png`

### TC-17 Просмотр пустого списка
**Описание:** `view` при отсутствии задач.  
**Предусловия:** Нет задач у текущего пользователя.  
**Ожидаемый результат:**  
- Таблица с заголовком и разделителем, но без строк задач.  
**Скриншоты:** `Screenshots/TC-17-view-empty.png`

### TC-18 Проверка изоляции пользователей
**Описание:** Просмотр задач одного пользователя не должен показывать задачи другого.  
**Предусловия:** Два разных пользователя с задачами.  
**Последовательность действий:**
1. Войти под пользователем A, добавить задачу.
2. Выйти, войти под пользователем B, выполнить `view`.
**Ожидаемый результат:** У пользователя B не отображается задача пользователя A.  
**Скриншоты:** `Screenshots/TC-18-isolation.png`

## 4. Обновление задач (update / status)

### TC-19 Обновление текста задачи
**Описание:** Изменение текста существующей задачи.  
**Предусловия:** Есть задача с номером 1.  
**Последовательность действий:**
1. `update 1 "Новый текст"`.
**Ожидаемый результат:**  
- Сообщение "Задача обновлена: 'Новый текст'".  
- При `read 1` текст изменён.  
**Скриншоты:** `Screenshots/TC-19-update.png`

### TC-20 Обновление статуса задачи
**Описание:** Изменение статуса.  
**Последовательность действий:**
1. `setstatus 1 InProgress`.
**Ожидаемый результат:**  
- Статус задачи меняется.  
- В `read` отображается новый статус.  
**Скриншоты:** `Screenshots/TC-20-setstatus.png`

### TC-21 Обновление несуществующей задачи
**Описание:** Попытка обновить задачу с неверным номером.  
**Последовательность действий:**

**Ожидаемый результат:**  
- Исключение "Задача с номером 999 не найдена."  
**Скриншоты:** `Screenshots/TC-21-update-notfound.png`

### TC-22 Обновление статуса несуществующей задачи
**Описание:** Аналогично для `setstatus`.  
**Ожидаемый результат:** Аналогичная ошибка.  
**Скриншоты:** `Screenshots/TC-22-status-notfound.png`

### TC-23 Обновление с некорректным статусом
**Описание:** `setstatus 1 InvalidStatus`.  
**Ожидаемый результат:** Исключение о неверном статусе.  
**Скриншоты:** `Screenshots/TC-23-invalid-status.png`
