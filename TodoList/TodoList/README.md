# TodoList Console App
Консольное приложение для управления списком задач.
## Возможности
- add <текст> — добавить задачу  
- view — показать список задач  
- done <индекс> — отметить задачу выполненной  
- delete <индекс> — удалить задачу  
- update <индекс> "новый текст" — изменить текст задачи  

## Хранение данных
Используются три массива одинаковой длины:
- todos — текст задач
- statuses — логический статус (выполнено / нет)
- dates — дата последнего изменения
Все массивы расширяются автоматически при необходимости.

TodoList (C# Console App)

Консольное приложение TodoList, написанное на языке C#.
Проект выполнен в рамках практического задания и работает без использования коллекций List<T>, данные хранятся в массивах.

Возможности:
- добавление задач
- многострочный ввод задачи
- просмотр списка задач
- табличный вывод с флагами
- просмотр полного текста задачи
- обновление задачи
- отметка задачи как выполненной
- удаление задачи
- обработка ошибок ввода

Запуск проекта:
dotnet run

Проект запускается из папки, где находится файл .csproj.

Команды:

add <текст>
Добавить задачу

add --multiline
Многострочный ввод задачи.
Вводите текст построчно, завершение ввода — !end

view
Показать список задач

Флаги команды view:
-i, --index        показать индекс задачи
-s, --status       показать статус задачи
-d, --update-date  показать дату изменения
-a, --all          показать все столбцы

read <id>
Показать полный текст задачи

update <id> <текст>
Обновить текст задачи

done <id>
Отметить задачу как выполненную

delete <id>
Удалить задачу

help
Показать список команд

Реализация:
- данные хранятся в массивах
- массивы автоматически расширяются
- используется switch для обработки команд
- реализованы проверки null и TryParse

## Практическая работа №7 — TodoList (паттерн Command)

В рамках практической работы №7 приложение переработано с использованием
паттерна проектирования **Command**.

### Архитектура
- Использован паттерн **Command**
- Каждая команда реализована отдельным классом, реализующим интерфейс `ICommand`
- Главный цикл программы не содержит логики команд
- Разбор пользовательского ввода вынесен в класс `CommandParser`
- Команды получают необходимые зависимости (`TodoList`, `Profile`) через параметры
